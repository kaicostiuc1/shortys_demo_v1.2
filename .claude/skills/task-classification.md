# Task Classification — Required First Step

Before writing or modifying ANY code in this project, you must declare which task type this is. Do not skip this step.

## Task Types

### ADDITION
Adding something new that does not currently exist.
- New component, new section, new nav item, new feature
- Rule: Touch ONLY the insertion point and nothing surrounding it. Do not refactor adjacent code.

### INSERTION
Placing a small element inside an existing component.
- Adding a div, a line, a prop to something that already exists
- Rule: Identify the exact line(s) to insert at. Change nothing else in that component.

### FIX
Something is broken and needs to be repaired.
- Rule: Diagnose first. State what is broken and why before touching any code. Fix only the broken logic. Do not use a fix as an opportunity to refactor, restyle, or reorganize.

### UPDATE
Changing existing content — copy, prices, menu items, hours.
- Rule: Find and replace the specific value. Do not touch surrounding structure or styles.

### FEATURE
A larger new capability spanning multiple components.
- Rule: Propose a plan first. Get approval before writing code. Build in isolation where possible.

## Enforcement

If you are unsure which type applies, ask before proceeding.
Never begin coding before declaring the task type.
Never perform a type of change broader than the declared task type.
